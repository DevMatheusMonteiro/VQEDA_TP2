# Análise de Funções Hash e Distribuição

Considerando uma tabela hash de tamanho M igual 10 e com chaves de números inteiros não negativos, vamos avaliar a sua distribuição, riscos de colisões e sua qualidade nas seguintes funções hash:

- `h(k) = k % M`
- `h(k) = (k * 7) % M`
- `h(k) = (k²) % M`

## `h(k) = k % M`

### Características

- Retorna o último dígito da chave
- Extremamente simples e rápida
- Boa distribuição quando os últimos dígitos das chaves são variados.
- Se todas as chaves possuírem o mesmo dígito (ex: 10, 20, 30, 40), todas colidem.
- Para conjuntos sequenciais, a distribuição é razoável, porém, previsível.
- Para chaves de 0 a 19:
  - De 0 a 9 cada um mapeia para si sequencialmente na mesma ordem
  - Ao chegar em 10 a 19, mapeia novamente de 0 a 9.

## `h(k) = (k * 7) % M`

### Características

- Multiplica por 7 e comprime com 10
- Produz distribuição completa de 0 a 9
- Produz todos os resultados possíveis 10 sem repetição
- Para chaves com o último dígito de 0 a 9, produz os seguintes resultados:
  `0, 7, 4, 1, 8, 5, 2, 9, 6, 3`
- Distribui melhor que a função hash anterior, pois "espalha" melhor, tendo um menor risco de colisão

## `h(k) = (k²) % M`

### Características

- O quadrado de números inteiros possui um forte padrão de acordo com o último dígito
- Para chaves com último dígito de 0 a 9, o último dígito de k² possui a seguinte sequência:
  `0, 1, 4, 9, 6, 5, 6, 9, 4, 1`
- A função não produz todos os resultados possíveis de 0 a 9
- Seus valores possíveis são apenas: `0, 1, 4, 5, 6, 9`
- Seu risco de colisão é o mais alto dentre as 3 funções
- Sua distribuição é muito ruim

# Conclusão

- Dentre as 3 funções a com menor risco de colisão e melhor distribuição é a `h(k) = (k * 7) % M`.
- `h(k) = k % M` é razoável, não distribui tão bem quanto a `h(k) = (k * 7) % M`.
- `h(k) = (k²) % M` é a pior das 3.
